# UX設計：オンエディット・アシスタントのコアループ

> 対象プロダクト: 図面変更履歴（△）を活用したCAD内AIアシスタント
> 関連: [企画コンテキスト](../../transcripts/2026-07-05_cad-ai-assistant-planning.md) / [フランジ変更履歴モック](../mocks/flange-revision-history.md)
> 方針: アプリUIラベルは**英語**、本ドキュメントはチーム共有用のため日本語。デザインは**ダーク基調のCADツール系**。
> 作成日: 2026-07-05

---

## 1. 体験の一言まとめ

> **設計者が図面を編集した瞬間、AIが必ず反応する。** 危険な変更なら過去の失敗を根拠に止め、通常の変更なら理由を先回りで下書きして「承認するだけ」で△履歴に残す。

差別化の核 ＝ **面倒な変更履歴の作成をAIが肩代わりする**。設計者は「書かない・承認するだけ」。単なるバージョン管理ではなく、Company Brainに溜まった過去知見を"編集の瞬間"に還元するプラットフォーム。

---

## 2. コアループ（統合オンエディット体験）

1つのトリガー（＝図面を編集した瞬間）から、AI体験が2つに枝分かれする。設計者から見れば「編集したら必ずAIが何か言ってくれる」一貫した1体験。

```
① You edit the drawing
      ↓
② Change auto-detected（差分検知：例 Thickness 3.5 → 4.0 mm）
      ↓
③ AI checks Company Brain（過去△知見と照合）
      ├─🔴 Risky change  → コミット前に警告（ブロッキング）
      └─🟢 New change    → 理由を推測して確認
      ↓
④ AI asks you to confirm（Approve / edit / reject）
      ↓
⑤ Logged as △ revision（理由付きで自動記録）
```

---

## 3. 2つの分岐（決定済みインタラクション）

### 🔴 分岐A：リスク警告 ＝ **ブロッキングモーダル**（デモの核心）
- **発火条件**: 編集内容が過去の△・再発パターンと一致（例: R部ダラし R5→R3）。
- **UI**: 危険な編集の瞬間に**モーダルで割り込み、確認するまで先に進めない**。
- **モーダルの中身**:
  - 何が問題か（"R部のダラしは過去に剛性低下を起こしています"）
  - 根拠となる過去△（2015 △2 → △3で手戻り、をタイムライン片で提示）
  - 推奨代替案（"ダラしではなく逃がし形状（2020 △3で成功）"）
  - アクション: `Apply recommendation` / `Proceed anyway` / `Ask 佐藤さん`
- **狙い**: 「同じ失敗の再発を編集の瞬間に止める」を強いインパクトで見せる。

### 🟢 分岐B：理由キャプチャ ＝ **候補から選択**
- **発火条件**: リスクなしの通常変更（例: Thickness 3.5 → 4.0 mm）。
- **UI**: AIが差分から理由を推測し、**複数の候補を提示 → ユーザーは1つ選ぶだけ**。
  - 例: `◉ Insufficient strength (FEM stress)` / `○ Weight reduction` / `○ Manufacturability` / `○ Other…`
  - 選択後、そのまま△エントリの下書きが完成 → `Confirm` で記録。
- **狙い**: 「書かない・選ぶだけ」で履歴が残る手軽さ。推測が外れても候補選択でリカバリしやすい。

## 4. UIステート（アシスタントパネル）

| # | ステート | 表示 |
|---|---|---|
| S0 | Idle | 直近の△・部品コンテキスト、設計者からの能動質問も受付 |
| S1 | Change detected | 差分チップ（変更前→変更後）を表示 |
| S2 | AI analyzing | Company Brain照合中の短いインジケータ |
| S3a | Risk warning | ブロッキングモーダル（分岐A） |
| S3b | Reason capture | 理由候補リスト（分岐B） |
| S4 | Confirmed | △エントリ生成、タイムラインに追記 |

## 6. 90秒デモシナリオ（体験の流れ）

1. **0–15s**｜ヒーロー画面。設計者が2025年フランジを開いている。右に△タイムライン。
2. **15–35s**｜通常変更（Thickness 3.5→4.0mm）を実施 → AIが理由候補を提示 → ワンタップ選択で△が自動記録（分岐B＝手軽さ）。
3. **35–65s**｜次に R部ダラし R5→R3 を編集 → **ブロッキングモーダルが割り込む**（分岐A）。2015→2025の再発を根拠提示。「逃がし形状」を推奨。
4. **65–85s**｜`Apply recommendation` で回避。△タイムラインに再発が"起きなかった"ことが残る。
5. **85–90s**｜締め：「過去の失敗を、次の設計者が繰り返さない」＝Company Brainの価値。

## 7. デザイントークン方針（ダークCAD系）

- **ベース**: ダークグレー基調（CAD作業画面に馴染む低輝度背景）。
- **アクセント**: 警告＝レッド系（分岐A）、確認/AI＝ティール or パープル系、記録済み＝グリーン系。
- **タイポ**: 英語UI。等幅を寸法・差分表示（3.5 → 4.0 mm）に使用。
- 既存の [design/index.html](../index.html) の仮パレット（青系ライト）は本トーンへ刷新する。
