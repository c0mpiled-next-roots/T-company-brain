# デモ運用手順書 — SenpAI(90秒デモ)

## 起動

```bash
cd app
npm install        # 初回のみ
npm run dev        # http://localhost:5173/?demo=1 を開く
```

- **本番URL**(Vercelデプロイ後): `https://<project>.vercel.app/?demo=1`
- **回線断フォールバック**: `npm run build && npm run preview` + スマホテザリング

## Claude API キー(ライブ演出用・任意)

- ローカル: `app/.env.local` に `ANTHROPIC_API_KEY=sk-ant-...`(gitignore済み。コミット厳禁)
- Vercel: ダッシュボード → Settings → Environment Variables に同名で設定
- **キーが無くても全シーン動作する**(確認質問・抽出は自動でスクリプト演出に降格)

## 90秒デモの操作(ビート表)

| 秒 | 操作する人 | 操作 | 起きること |
|---|---|---|---|
| 0-15 | 発表者 | (スライドで問題提起) | — |
| 15-40 | 操作者 | **「R3」ボタンをクリック** | 先輩カードが割り込む(R2発火)。R部が赤く点滅 |
| 40-60 | 操作者 | 「⚙ 知識コンパイラ」→「▶ コンパイル」 | 報告書→構造化レッスンJSONがタイプされる →「🧠 脳に追加」 |
| 60-80 | 操作者 | 先輩カードの**「✓ 逃がし形状を適用(推奨)」**→ 質問に**「はい、そうです」** | 緑の逃がし形状がモデルに付く → 確認質問 → グラフに新ノード、🧠カウント+1 |
| 80-90 | 発表者 | (締めのスライド+QR) | — |

> 注意: シーン順の都合で、コンパイラ(40-60)を先輩カードを閉じずに開くとカードが背面に残る。
> リハーサルでは 15-40 のカード表示中に「⚙ 知識コンパイラ」を開き、閉じてからカードの適用ボタンを押す流れを体で覚えること。

## 絶対にやってはいけないこと

- **板厚スライダーとR3ボタン以外のパラメータ操作をしない**(発火はR1/R2のみ=偽陽性ゼロ設計。意図しない操作で「警告が出ない」場面を見せると価値提案が崩れる)
- デモ中のページリロード(グラフの成長がリセットされる)

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 警告が出ない | すでに同ルール発火済み(1セッション1回)。ページをリロードして最初から |
| 確認質問/抽出が遅い | 4〜9秒でスクリプト演出に自動降格するのでそのまま待つ |
| 画面が真っ白 | `npm run build && npm run preview`(ポート4173)に切り替え |

## 審査Q&A(想定問答はスライド最終ページ)

- 「実CADに組み込めるの？」→ 本番はCADアドインAPI(CATIA/NX/SolidWorks全てが公式に提供)経由。今日のWebビューアはデモ用の器
- 「△の注記だけで情報足りるの？」→ △はリンクキー。報告書・PPTが知識の本体(コンパイラのシーンがまさにそれ)
- 「GleanやCoLabと何が違う？」→ 既存ツールは形式化済み知識を「読む」だけ。うちは非形式知をコンパイルし、さらに設計行為から知識を「書き込む」フライホイールを持つ
