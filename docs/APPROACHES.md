# APPROACHES — 今日の実装アプローチ 3案 + 組み合わせ

/office-hours セッションの Phase 4(代替案)。この中から1つ選ぶと設計ドキュメントに確定される。

前提: デモの核は「設計操作の瞬間に先輩AIが根拠付きで警告する」体験([PREMISES.md](PREMISES.md) 参照)。
どの案でも警告の照合は **LLMではなく決定的ルールマッチ**(幻覚ゼロ・遅延ゼロ)で行い、
LLMの役割は案ごとに異なる。

---

## APPROACH A: 全静的ワンページ(最小構成)

**概要**: 静的HTML + three.js(CDN読み込み)だけで作る。知見約30件を手書きJSONに構造化し、
決定的ルールマッチで警告。先輩カードの文言も事前作成。**実行時にLLMは一切動かない**。
デプロイは GitHub Pages / Vercel 静的ホスティング。

| 項目 | 評価 |
|---|---|
| Effort | S (human: ~1日 / CC: ~1時間) |
| Risk | Low |
| Completeness | 5/10 |

- ✅ 絶対に動く。失敗モードほぼゼロ
- ✅ CLAUDE.md の「静的HTML・ブラウザで直接開ける」方針そのまま
- ✅ 会場Wi-Fi非依存。デプロイ一瞬
- ❌ 実行時にAIが動いていない=「AIプロダクトです」の主張が弱い
- ❌ 「散在知識→構造化」のコンパイラ部分が口頭説明のみになる

**Reuses**: design/mocks/flange-revision-history.md、presentation/、design/

## APPROACH B: フルスタック + LLM演出(理想形)

**概要**: Vite/Next.js アプリを Vercel にデプロイ。中身は:

1. **three.js ビューア** — フランジ3Dモデル + パラメータ操作(板厚/R/穴位置)
2. **決定的マッチ警告** — 操作の瞬間に先輩カード(過去の△・出典・担当者・再発履歴)
3. **Claude API ①** — 照合済み知見の上に自然言語の説明文を生成(先輩の口調)
4. **Claude API ② 「コンパイラ」ライブシーン** — トラブル報告書テキストを画面に投入すると、
   構造化レッスンJSON(category/部位/操作/outcome)が**目の前で生成される**(15秒)
5. **知識グラフ表示** — 部品⇄△⇄文書⇄人を結ぶ「生きた地図」の絵

| 項目 | 評価 |
|---|---|
| Effort | M (human: ~1週間 / CC: 今日の5時間の大半) |
| Risk | Medium(APIキー・デプロイ・会場ネットワーク依存) |
| Completeness | 7/10 |

- ✅ 差別化主張(コンパイラ)が口約束でなく目の前で動く
- ✅ 「生きた地図」がそのまま絵になる。Biggest Engineering Lift 賞も狙える
- ✅ 審査員が触れるライブURL
- ❌ 5時間での失敗モードが増える(API・デプロイ・ネットワーク)
- ❌ デモ中のネットワーク障害リスク(→抽出シーンは事前録画のバックアップを用意)

## APPROACH C: フライホイール実演(創造的・終盤ビート)

**概要**: A または B の上に「**書き込みパス**」を追加する。デモの終盤:

1. 設計者が警告を受け入れて設計変更を確定する
2. AIが「**この変更の理由は強度不足への対応ですよね？**」とワンタップ確認質問
3. 設計者が「はい」を押す
4. **知識グラフに新しいノードがその場で増える** —「脳が今、育った」

| 項目 | 評価 |
|---|---|
| Effort | +S (A/Bに追加。human: ~2日 / CC: ~45分) |
| Risk | Low-Med |
| Completeness | 組み合わせ次第 |

- ✅ 競合(CoLab/Leo/CADDi)が誰も持たない「living map が自己更新する」瞬間の実演
- ✅ 90秒デモの完璧な締め。「使うほど賢くなる」=データ・フライホイール=投資家向けモート
- ✅ 将来ビジョン(設計行為そのものが知識化される世界)の証明
- ❌ スコープ追加。演出を厳密にスクリプト化する必要

## 組み合わせの選択肢

| 選択肢 | 内容 | Completeness | 向いている場合 |
|---|---|---|---|
| **B + C** ⭐推奨 | フル機能帯 + フライホイール締め | 9/10 | 優勝と投資可能性の両狙い。チームで分担できる場合 |
| B のみ | 読み取りパス集中 | 7/10 | フライホイールはスライドで語る。完成度重視 |
| A + C | 静的 + フライホイール演出 | 7/10 | リスク最小で物語は完結させたい場合 |
| A のみ | 最小構成 | 5/10 | 残り時間をプレゼンとリハに全振りしたい場合 |

## 推奨: B + C

理由: ゴールは「優勝 + 起業視野」。優勝には B のライブ抽出と地図の絵、
投資可能性には C のフライホイールが直撃する。

**退避ラダー**(時間が押したら段階的に降格):

1. B+C フル → 2. B(抽出シーンを事前録画に差し替え) → 3. A+C → 4. A

締め切り90分前の時点で動いているものが「今日の製品」。そこから先は磨きとリハーサルのみ。
