Observed inside a real automotive design floor. This is not a hypothesis.

Why Now

The knowledge was never missing.
It just couldn't be executed.

Every existing tool only checks knowledge that is already formalized into a company standard — because turning messy documents and revision notes into executable rules was technically impossible.

Large language models change that. For the first time, unstructured design knowledge can be compiled into live, executable checks. That is the whole product.

The EUREKA: existing tools handle only formalized knowledge because the conversion couldn't be done — until now.

How It Works

A compiler from tribal knowledge
to executable design checks.

01 · MATCH

Deterministic rules

Matching is rule-based, not LLM — zero hallucination, zero latency, zero false positives. False positives would kill this product.

02 · COMPILE

LLM as compiler

Reports & △ notes → structured, executable lessons. The LLM does the conversion no one could do before.

03 · MAP

Knowledge graph

Part ⇄ △ ⇄ document ⇄ person — a living map of how the company actually designs.

The senpai card is deterministic and pre-generated, so it never depends on the network to fire in 2 seconds.

APPENDIX · Q&A Anticipated Questions

Defensible answers.

  • "Can it plug into real CAD?" — Via CAD plug-in APIs (CATIA / NX / SolidWorks all expose add-in APIs). Explicitly out of scope for today.
  • "What about false positives?" — On stage only R1/R2 fire, matching is deterministic. No off-target warning is ever shown.
  • "Who are the competitors?" — CoLab AutoReview (closest, read-only) · Leo AI · CADDi · DraftAid. None have the write loop.
  • The Assignment — Collect 50 real drawings, measure how many △ notes alone yield an executable check. That number decides where we stand vs. CADDi / CoLab.

Backup ladder: full live → recorded extraction → typewriter pseudo-animation. A working build always runs locally + tethering.
